# Audit Fix Progress

**Audit source:** `data/audit-2026-04-25/index.md` — 135 findings from the pre-release UX/product audit. **Session date:** 2026-04-28 **Final tally:** 🟢 123 Done · ⚪ 4 Open · 🟡 7 Deferred · ✱ 1 Skipped

**#136 — @ mention in the markdown editor not working.** Added as a new audit finding. The status bar hinted `@` mention but no mention extension exists in the editor. The hint has been removed to avoid misleading users. Implementation tracked as #136 in the audit.

**#137 — Navigating to Agents/Tasks loses the open KB page.** When users leave the editor (go to Agents or Tasks section) and use the back/return chip to come back, the editor should re-show the page they had open. Added as new finding #137 in the audit. `document.title` now also reflects the current page/section name so the tab gives context.

**#138 — Tasks schedule tab had a Calendar/List toggle.** The list subview added unnecessary complexity — the calendar is the primary affordance. Calendar-only view is now the default; list subview and its toggle button have been removed from `schedule-view.tsx`.

## Today's active code changes

These are the items that had **new code written** on 2026-04-28, grouped by commit.

### Commit `7dd9de7` — batch of 10 UX polish fixes

**#022 — Sidebar shortcuts discoverable.** Cmd+1/2/3 already wired to switch Data/Agents/Tasks drawers. Added them to each tab's `aria-label` and `title` so users can discover them without hunting through docs.

**#139 — Dynamic document title (chrome tab).** `document.title` now updates on every section/page change: page filename when in the editor, agent slug when on an agent detail, "Tasks", "Agents", "Settings", etc. for top-level sections. All formatted as `Name — Cabinet`. Running task count animation left for a follow-up (needs task-store subscription).

**#031 — Editor toolbar overflow.** The "More ⋯" overflow popover has been replaced with a single scrollable row. All secondary buttons (alignment, super/sub, media, undo/redo, RTL) are now appended to the scrollable row after the primary set. Gradient fade arrows on the left/right edges signal that more items exist off-screen; the native scrollbar stays hidden via `scrollbar-none`.

**#034 — Page H1 too small.** Breadcrumb leaf font-size bumped 13 px → 15 px; ancestors tightened to 11 px. Current page title now reads as an H1, not a breadcrumb sibling.

**#040 — No autosave indicator.** New three-state indicator: "Saving…" + spinner while debounce pending; "Saved ✓" (green) for 2 s after write; nothing while idle. Removes the noisy always-visible "Ready" state.

**#041 — Toolbar buttons missing active state.** Active buttons (Bold, Italic, Heading, etc.) now get `ring-1 ring-inset ring-foreground/20` — an unmistakable inset ring that doesn't rely on color alone.

**#047 — Agents tab blank flash.** Agents overview renders 3 skeleton cards while loading instead of flashing the empty state. The flash lasted ~200 ms but felt broken.

**#063 — YOUR TURN badge.** Cards waiting for input show "Approval" (violet + ShieldCheck) when `pendingActions` present, or "Question" (amber + MessageCircleQuestion) for free-text requests. Previously both used the same neutral pill.

**#064 — RUNNING column collapsed by default.** Force-expanded while items > 0; collapse button hidden. A running task is the most time-sensitive thing on the board — it should never be hidden.

**#066 — Compact-rows toggle no effect.** Density toggle icon shows the *target* state (Rows4 = will go compact; Rows2 = will go comfortable) and gained a text label. Previous icon read as a status indicator, not a button. Note: `density` prop is available to `ScheduleView` via `tasks-board.tsx` but the calendar component doesn't yet consume it — a follow-up to the schedule calendar component is needed.

**#110 — Mixed verb casing on Add buttons.** Sidebar drawer tooltips normalised to "New Page / New Agent / New Task"; agents workspace "Add agent" → "New Agent". All creation CTAs now follow the same `New [Noun]` pattern.

### Commits `0704983 → 9fe2cf0` — hotkeys + inbox composer

**#130 — "+ New Task" routes then opens dialog.** Now opens `AddToInboxDialog` as a floating overlay in-place. Previously navigated to the tasks route first, then opened the dialog — disorienting if you were on any other page.

**#135 — Global shortcuts conflict with system keys.** ⌘N (new window), ⌘M (minimize), ⌘\` (cycle windows) were all hijacked. Replaced: new task is ⌘⌥T, mute is ⌘⌥M, terminal is ⌘⌥\`. A help slide was added showing the full shortcut map.

### Afternoon conversation session (uncommitted)

**#054 — Heartbeat pause buried under Manage.** Heartbeat row in agent detail now shows an inline on/off toggle. Green "on" → gray "off" on click; calls `PATCH /api/agents/:slug` to flip `active`. Run Now disabled while off. Per-agent toggle is the on/off pill in the heartbeat row. A "Pause all / Resume all" button for the whole cabinet is in the Agents workspace Heartbeats section header.

**#076 — Cmd+K has no recent items.** Search palette stores last 5 queries + page IDs in `localStorage`. On re-open, recent list shown before user types anything.

**#078 — Empty search no suggestions.** First-open (no recents yet): palette shows "Search pages, agents, and tasks" + three example hint lines so the empty state doesn't look broken.

**#088 — About version may not match release file.** Settings About tab now imports `cabinet-release.json` directly. Version + channel badge read from the same file the daemon uses — no more drift.

**#101 — Focus rings hard to see in light theme.** Global `* { outline: ring/50 }` was setting outline opacity to 50%. Added `:focus-visible { outline-color: var(--ring); }` in `@layer base` to restore full-opacity rings on keyboard focus only.

**#113 — Tour copy out of date.** Agents slide rewrote "a persona, a heartbeat, and scheduled jobs" → "a persona, its own schedule, and a memory." Drops the unexplained "heartbeat" jargon on first encounter.

**#117 — "did not run" rows lack inline Run-now.** `EventDot` tooltip adds amber "Click to run now →" when `missed`. `EventPill` title updated to "did not run — click to run now". Affordance was already there; users just had no way to discover it.

**#119 — Heartbeats can't be muted per cabinet.** "Pause all / Resume all" button added to Agents workspace Heartbeats section header (the area above the agent card list, next to the "Heartbeats" section title). Calls `POST /api/agents/scheduler` with `{ action: "stop-all" | "start-all", cabinetPath }`. Backend already supported this; UI didn't surface it.

**Bonus — Inbox empty state → clickable card.** The dashed INBOX placeholder in the kanban is now a full `<button>` that calls `onAddTask`. Hover brightens border, tints background, darkens copy. Keyboard-navigable with a full `focus-visible` ring. Shortcut hint updated to "Click or press ⌘⌥T".

**#140 — "Add task" always visible in Inbox column.** A subtle "+ Add task" button is now rendered at the bottom of the Inbox column even when tasks already exist, so users don't have to use the keyboard shortcut to add more.
